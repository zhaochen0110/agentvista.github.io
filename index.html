AgentVista: Evaluating Multimodal Agent in Ultra-Challenging Realistic Visual Scenarios

Abstract

Real-world multimodal agents solve multi-step workflows grounded in visual evidence. For example, an agent can troubleshoot a device by linking a wiring photo to a schematic and validating the fix with online documentation, or plan a trip by interpreting a transit map and checking schedules under routing constraints. However, existing multimodal benchmarks mainly evaluate single-turn visual reasoning or specific tool skills, and they do not fully capture the realism, visual subtlety, and long-horizon tool use that practical agents require. We introduce AgentVista, a benchmark for generalist multimodal agents that spans 25 sub-domains across 7 categories, pairing realistic and detail-rich visual scenarios with natural hybrid tool use. Tasks require long-horizon tool interactions across modalities, including web search, image search, page navigation, and code-based operations for both image processing and general programming. Comprehensive evaluation of state-of-the-art models exposes significant gaps in their ability to carry out long-horizon multimodal tool use. Even the best model in our evaluation, Gemini-3-Pro with tools, achieves only 27.3% overall accuracy, and hard instances can require more than 25 tool-calling turns. We expect AgentVista to accelerate the development of more capable and reliable multimodal agents for realistic and ultra-challenging problem solving.

Dataset Construction Pipeline

AgentVista is built from 300k+ real images through a rigorous 4-stage pipeline: (1) Agent-centric filtering reduces the pool to 568 candidates (0.19%); (2) Expert finalization produces 315 tasks; (3) Execution filtering retains 241 tasks with verified tool-use diversity; (4) Two-round verification yields the final 209 tasks. On average, constructing a single instance takes about 4 hours.

Overview of the AgentVista dataset construction pipeline, consisting of agent-centric filtering, expert finalization, execution filtering, and two-round verification.

Analysis

Tool-use distribution across models. GPT models rely more on the code interpreter, while Gemini and Claude models use web search most frequently.

Error category distribution across four multimodal models. Visual misidentification is the dominant failure mode across all models, indicating that many errors originate from incorrect grounding on fine-grained visual evidence.

Tool ablation on Gemini-3-Pro and Claude-Sonnet-4.5. Both models perform best with the full tool suite, highlighting the importance of combining visual manipulation and retrieval.

Image manipulation operation distribution of code interpreter calls across four models. Crop is the most frequent operation, suggesting that many interactions rely on localized visual grounding before further reasoning.

Case Studies

We present representative agent trajectories from Gemini-3-Pro to illustrate both successful and failed problem-solving patterns. Each case shows the input query, tool-calling steps, and final outcome.

Success Gemini-3-Pro · 5 tool calls
Task: Acting as a luxury sneaker authenticator, search for authentic craftsmanship images for comparison. Decide whether the shoes are Real or Fake, and give at least two visual reasons.

Input Images

Ground Truth: Counterfeit. Key cues: Widow's Peak Error, Poor Tongue Font & Stitching, Misaligned 3M Stripes.

Tool Trajectory

Step 1
web search Retrieved legit-check guides highlighting tongue tag font, stitching, and interior label formats.
Step 2
image search Collected authentic reference images for tongue tag layout and font.
Step 3
image search Collected authentic inside label formats and placement references.
Step 4
web search Found authentic style-ID patterns; generic sticker codes indicate replicas.
Step 5
image search Confirmed generic interior stickers are associated with replicas.
Final Answer: Fake. The interior sticker code format is inconsistent with authentic labeling; the tongue tag font and stitching do not match authentic references. ✓ Correct
Success Gemini-3-Pro · 7 tool calls
Task: From the beers shown in the photo, consider only German-brewed beers with ABV > 5%. Which brand has the highest total alcohol per can, accounting for both ABV and can volume?

Input Image

Ground Truth: Tie: Steam Brew German Red and Perlenbacher Strong (both 7.9% ABV x 500ml = 39.5ml alcohol).

Tool Trajectory

Step 1
code interpreter Zoomed and cropped the photo to read brand names and ABV/volume on cans.
Step 2
web search Retrieved product pages confirming both beers are 7.9% ABV.
Step 3
code interpreter Cropped individual cans to verify German origin and confirm can capacity.
Steps 4-7
web search + image search Cross-checked ABV values, confirmed 500ml can size, verified no higher-ABV German option among eligible beers.
Final Answer: Tie: Steam Brew German Red and Perlenbacher Strong. Both are German-brewed, exceed 5% ABV, and have the same alcohol per can: 0.079 x 500ml = 39.5ml. ✓ Correct
Failure Gemini-3-Pro · 3 tool calls
Task: The first image shows a 6x6 jigsaw puzzle (one piece missing) made from a karst-landform photo. The second image is a local reference landscape. Reconstruct the puzzle and report the coordinate of the missing piece.

Input Images

Ground Truth: (3, 1)

Tool Trajectory

Step 1
code interpreter Attempted to segment the puzzle into individual pieces. The segmentation merges adjacent pieces and fails to recover all pieces.
Step 2
code interpreter Improved segmentation with morphological operations. Still under-segments: extracts only 24 regions instead of the expected 35 pieces.
Step 3
code interpreter Attempted to visualize extracted pieces for matching. Confirmed missing segments — reconstruction cannot proceed.
Failure Point: Step 2 (segmentation). The model cannot reliably separate touching pieces, blocking the entire downstream workflow. ✗ Failed
Failure Gemini-3-Pro · 11 tool calls
Task: A colleague referenced this venue only as "Target Arena A." Identify exactly which university facility is shown so the correct team metadata can be linked in a database.

Input Image

Ground Truth: UNC

Tool Trajectory

Step 1
code interpreter Inspected the scene and located identifiers: focused on floor logo for identification.
Step 2
code interpreter Cropped the court logo and scoreboard. Produced a small, low-detail crop.
Steps 3-4
image search + code interpreter Reverse searched the crop (no matches). Re-cropped with more context — still ambiguous.
Steps 5-7
image search + web search + code interpreter Reverse search again failed. Used structural cues (exposed truss ceiling, track). Enhanced logo contrast — still couldn't resolve mascot identity.
Steps 8-11
image search Pivoted to guessing University of St. Thomas AARC based on weak visual similarity. Over-weighted generic ceiling/track features rather than definitive identifiers. Committed to incorrect university.
Failure Point: Step 9 (incorrect anchoring). After failing to identify the floor logo, the model anchored on St. Thomas using generic architectural similarities rather than definitive visual evidence. ✗ Failed
